# QueueFlow 360 – Smart Queue, Appointment & Customer Service Management Platform

QueueFlow 360 is a full-stack web application designed for seamless customer service management. It enables customers to book appointments, generate live queue tokens, and monitor their status in real-time. System administrators can manage departments, services, live queues, and view analytics via a comprehensive dashboard.

## Technology Stack

- **Backend:** Java 21, Spring Boot 3, Spring Web, Spring Data JPA, Spring Security (JWT)
- **Frontend:** React.js 18 (Vite), React Router DOM, Axios, Bootstrap 5, Custom CSS (Glassmorphism)
- **Database:** MySQL
- **Build Tools:** Maven (Backend), npm (Frontend)

## Installation & Setup

### 1. Database Setup
1. Install and start **MySQL**.
2. Create a database named `queueflow360`:
   ```sql
   CREATE DATABASE queueflow360;
   ```
3. The tables will be automatically generated by Hibernate when the backend starts.
4. Open your MySQL client and execute the `schema.sql` file located in the root directory to insert the default Admin user, departments, and services.
   - **Admin Login:** `admin@queueflow360.com`
   - **Admin Password:** `admin123`

### 2. Run Backend (Spring Boot)
1. Open the `backend` folder in IntelliJ IDEA, Eclipse, or VS Code.
2. Verify database credentials in `backend/src/main/resources/application.properties`:
   ```properties
   spring.datasource.username=root
   spring.datasource.password=root
   ```
3. Run the application via your IDE or using Maven:
   ```bash
   cd backend
   mvn spring-boot:run
   ```
4. The backend will start on `http://localhost:8080`.

### 3. Run Frontend (React Vite)
1. Open a new terminal and navigate to the `frontend` directory.
2. Install dependencies:
   ```bash
   cd frontend
   npm install
   ```
3. Start the Vite development server:
   ```bash
   npm run dev
   ```
4. The frontend will be available at `http://localhost:5173`.

## API Documentation

### Authentication
- `POST /api/auth/register` - Register a new customer
- `POST /api/auth/login` - Authenticate and receive JWT

### Admin 
- `GET /api/admin/dashboard` - Get overall statistics
- `GET, POST, PUT, DELETE /api/admin/departments` - Manage Departments
- `GET, POST, PUT, DELETE /api/admin/services` - Manage Services

### Customer
- `GET /api/customer/departments` - View available departments
- `GET /api/customer/services/department/{id}` - View services
- `POST /api/customer/appointments` - Book appointment (generates Token)
- `GET /api/customer/appointments` - View appointment history

### Queue Management
- `GET /api/queue/status` - (Customer) View live token status
- `GET /api/queue/waiting` - (Admin) View waiting tokens
- `POST /api/queue/call-next` - (Admin) Move next token to IN_PROGRESS
- `POST /api/queue/complete/{token}` - (Admin) Complete a token
- `POST /api/queue/skip/{token}` - (Admin) Skip a token
